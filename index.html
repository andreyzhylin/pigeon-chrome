Вступление

Для чего служит pigeon?

В некоторых случаях JavaScript код должен правильно работать на разных веб сайтах (например расширение для браузера). В таком случае проверка правильности часто сводится к открытию страницы, выполнению скрипта в консоли и анализу полученного результата. Но самое неприятное заключается в том, что если алгоритм слегка меняется, то необходимо выполнять все проверки заново. Pigeon был создан с целью автоматизировать этот процесс.

Принцип работы

Pigeon реализован в виде расширения к браузеру Chrome и использует его API для запуска тестов. Для начала работы пользователю необходимо задать URL страницы на которой будут выполняться тесты и создать один или несколько тестов, которые будут на ней выполнены (подробнее об использовании в пункте "Пример использования").

При нажатии на кнопку запуска теста, Pigeon выполнит следующие действия:

  1. Откроет вкладку с заданной страницей

  2. Дождется загрузки страницы

  3. Выполнит служебный код

  4. Выполнит код пользовательских файлов (в порядке загрузки)

  5. Выполнит код теста

    • Если в ходе выполнения возникла ошибка или пользователем вызван метод pigeon.reject, то тест пройден с ошибкой
    • Если все вызовы метода pigeon.expect теста получили истинное значение, то тест пройден успешно
    • Иначе тест провален

Установка

При помощи пакета crx

  1. Скачайте последнюю версию.
  2. Зайдите на страницу расширений - chrome://extensions/ (Дополнительные инструменты > Расширения).
  3. Перетащите скачанный файл на страницу.

Из исходников

Установите Node.js и затем:

$ git clone https://github.com/andreyzhylin/pigeon-chrome.git
$ cd pigeon-chrome
$ npm -g install grunt-cli karma bower
$ npm install
$ bower install
$ grunt build
  • Зайдите на страницу расширений - chrome://extensions/ (Дополнительные инструменты > Расширения).
  • Убедитесь, что установлен флажок "Режим разработчика" в правом верхнем углу.
  • Нажмите "Загрузить распакованное расширение...".
  • Переместитесь к папке '*/pigeon-chrome/build' и выберите её

Пример использования

Базовое использование

  1. Откройте страницу "Параметры", вы увидите главную страницу приложения:

  2. Переключитесь для удобства на русский язык в правом верхнем углу:

  3. Перейдите на страницу "Настройки" и нажмите "Установить базовый набор":

  4. Перейдите на страницу "Обзор тестов", вы увидите, что появилась таблица с базовым набором тестов:

  5. Есть 2 основных сущности, это "страница" и "тест", страницы выделены синим цветом в таблице. Нажмите на иконку редактирования напротив первой страницы: Описание страницы - отображается в таблице, URL адрес - адрес на котором будут выполняться тесты. При отмеченном флажке Pigeon будет открывать новое окно вместо новой вкладки при запуске тестов - это делает вкладку теста активной и полезно при отладке или если DOM не обновляется на неактивной вкладке.

  6. Вернитесь к обзору и нажмите на иконку редактирования напротив первого теста первой страницы: Тест содержит описание, отображаемое в таблице, код, выполняемый на странице и флажок режима отладки, о котором будет речь в разделе "Отладка тестов".

  7. Перед запуском этого теста, разберем его код:

    var searchInput = document.getElementsByName('q')[0];

pigeon.expect('Поле ввода поиска определено', typeof searchInput !== 'undefined');
pigeon.expect('Тег поля ввода совпадает', searchInput.tagName === 'INPUT');

// Метод 'pigeon.resolve' должен вызываться, тест считается полностью выполненным
pigeon.resolve();

    У страницы был установлен URL - https://www.google.com/, поэтому после запуска откроется вкладка с этим адресом. После загрузки страницы начнет выполняться код. Тест называется "Страница загружается", поэтому данным кодом мы:

    • Проверяем что строка поиска (элемент DOM с именем 'q') присутствует на странице
    • Строка поиска является тегом 'input'
    • Завершаем тест

    В коде теста используются служебные методы pigeon:

    pigeon.expect(message, testCase)

pigeon.resolve()

    pigeon.expect - создает значение, которое должно быть проверено, если хотя бы один из expect методов примет значение testCase === false, то тест не будет пройдет, а в таблице будет указан message.

    pigeon.resolve - сообщает, что тест закончил выполнение.

    Вернемся к обзору тестов и запустим данный тест: Как мы видим тест завершился успешно, а значит все параметры переданные в pigeon.expect являются истинными значениями.

  8. Рассмотрим код следующего теста "Google находит сайт github.com":

    location.href = '/#q=github';
// Ждем 1 секунду, пока обрабатывается наш запрос

setTimeout(function () {
  pigeon.expect('Результаты загружены', typeof ires !== 'undefined');
  pigeon.expect('Адрес сайта присутствует в результатах', ires.innerText.indexOf('github.com') !== -1);

  // В данном случае тест считается законченным здесь, а не в конце скрипта
  pigeon.resolve();
}, 1000);

pigeon.expect('Результаты не появляются мгновенно', typeof ires === 'undefined');
// Здесь заканчивается выполнение скрипта, но тест будет выполнен
// после вызова pigeon.resolve или по таймауту

    В данном случае мы меняем свойство location.href, что приводит запросу поиска, т.к. на это потребуется время, то проверку будем проводить через 1 секунду, а сразу же проверим, что результат еще не готов.

    После этой проверки, как указано в комментарии, скрипт заканчивает работу, но т.к. метод pigeon.resolve не вызывался, тест на заканчивается.

    Через секунду вызовется функция проверки, в которой мы убедимся, что "Google находит сайт github.com" и вызовом метода pigeon.resolve завершим тест.

  9. Код третьего теста похож на код второго, но теперь мы ищем упоминания о pigeon-chrome:

    location.href = '/#q=pigeon-chrome';
// Ждем 1 секунду, пока обрабатывается наш запрос

setTimeout(function () {
  pigeon.expect('Результаты загружены', typeof ires !== 'undefined');
  pigeon.expect('Гугл не нашел pigeon :(', ires.innerText.indexOf('pigeon-chrome') !== -1);

  // В данном случае тест считается законченным здесь, а не в конце скрипта
  pigeon.resolve();
}, 1000);

    Запустите второй и третий тест и вы увидите, что второй завершился успешно, а третий не пройден и при клике на знак вопроса отображает message того метода pigeon.expect, который получил параметр false:

Обработка ошибок

Если код вашего теста генерирует ошибку, то тест помечается статусом "Ошибка" и снабжается сообщением этой ошибки.

На второй странице базового набора демонстрируются основные виды ошибок, которые могут возникнуть:

  1. Ошибки компиляции (Синтаксические ошибки)

    if { // Ошибка компиляции

pigeon.expect('Always success', true);
pigeon.resolve();

  2. Ошибки времени выполнения (Семантические ошибки)

    var foo = bar.baz;

pigeon.expect('Always success', true);
pigeon.resolve();

  3. Таймаут операции

    // Таймаут операции возникает, если метод pigeon.resolve не был вызван в течение заданного времени
// Вы можете изменить время на вкладке "Настройки" или отключить для любого теста посредством режима отладки

pigeon.expect('Всегда успешно', true);
// pigeon.resolve();

  4. Ошибка pigeon.expect

    // Метод pigeon.expect ожидает значение типа Boolean в качестве 2 параметра
// В случае если будет получено значение другого типа, возникнет ошибка
pigeon.expect('Ответ на главный вопрос жизни, вселенной и всего такого.', 42);
pigeon.resolve();

  5. Пользовательские ошибки

    pigeon.expect('Всегда успешно', true);

pigeon.reject('Описание пользовательский ошибки');

pigeon.resolve();

    Здесь использован третий служебный метод:

    pigeon.reject(message) - используйте его, если самостоятельно обрабатываете ошибки.

Отладка тестов

  1. Для отладки тестов необходимо установить флажок "Режим отладки" на странице редактирования теста. В этом режиме, тест не будет завершаться по таймауту, а вкладка останется открытой даже после завершения теста.

  2. Все вызовы служебных методов автоматически логируются в консоль (запустите 1 тест на 3 странице):

    // В режиме отладки вкладка не будет закрыта после выполнения теста или по таймауту
// Что позволяет вам отладить ваши тесты
pigeon.expect('Всегда успешно', true);
pigeon.expect('Всегда не пройден', false);
pigeon.expect('Не значение типа boolean', [1]);
pigeon.resolve();

  3. Вы также можете использовать в своём коде такие средства как console.log, debugger, и т.д. Рассмотрим код последнего теста:

    // Вы можете использовать панель разработчика и такие средства как console.log, debugger и другие
console.log('Начало теста');

var logo = document.getElementsByClassName('central-featured-logo')[0];
logo.onclick = function () {
  console.log('Клик по лого');
  debugger;
  pigeon.expect('Всегда успешно', true);
  pigeon.resolve();
}

    Выводим в консоль текст и подписываемся на событие клика по лого. После клика - выведем еще немного текста в консоль, остановимся в точке останова и после продолжения успешно завершим тест:
